# 楽天銀行 WEB-FB（総合振込）CSVファイル仕様

## フォーマット概要
- CSV形式（カンマ区切、可変長）
- 拡張子: `.csv`
- ダブルクォーテーション: あり/なし両方可

## 列構成（左から順に）

| 項目 | 文字 | byte数 | 内容 |
|------|------|--------|------|
| 1. サービス区分 | 半角数字 | 1byte | 全行に数値「3」を入力（総合振込の場合） |
| 2. 実行日 | 半角数字 | 4byte | MMDD形式（例：0320 = 3月20日） |
| 3. 受取人銀行番号 | 半角数字 | 4byte | 金融機関コード（4桁） |
| 4. 受取人支店番号 | 半角数字 | 3byte | 支店コード（3桁） |
| 5. 受取人預金種目 | 半角数字 | 1byte | 1=普通、2=当座、4=貯蓄 |
| 6. 受取人口座番号 | 半角数字 | 7byte | 口座番号（7桁、左詰め） |
| 7. 受取人口座名 | 全角カナ | 2-60byte | 口座名義（カタカナ） |
| 8. 金額 | 半角数字 | 1-10byte | 振込金額（円） |
| 9. 顧客番号 | 半角英数字 | 1-20byte | 任意（管理用） |

## サンプルデータ
```csv
3,0820,0191,100,1,0045683,タナカイチロウ,500,001
3,0820,0568,256,2,1055548,スズキサブロウ,1000,002
3,0820,0003,460,4,1456554,マツイタダシ,10000,003
```

## 注意事項
- 実行日は全行同一日にする必要がある
- 実行日は金融機関営業日のみ指定可能
- 口座名義は全角カタカナで入力
- 顧客番号は任意（出金申請IDなどを入れると管理しやすい）
